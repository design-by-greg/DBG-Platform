# ADR-004 — Workspace Model

**Status:** Accepted

## Context

Une Organisation peut avoir plusieurs besoins ou périmètres : communication générale, textile, événement, marque, saison sportive, projet commercial ou salon.

## Decision

Chaque Organisation peut posséder un ou plusieurs Workspaces.

Un Workspace est un espace de travail interne permettant de regrouper des projets, assets, commandes, documents et workflows liés à un périmètre donné.

## User-facing vocabulary

Le terme interne reste **Workspace**.

Dans l'interface client, le terme affiché pourra être **Projet** lorsque cela sera plus clair.

## Consequences

Cette architecture permet à une Organisation de gérer plusieurs contextes sans créer plusieurs comptes séparés.
