# Architecture Decision Records

Les ADR documentent les décisions d'architecture structurantes de DBG Platform.

## Format

Chaque ADR doit expliquer :

- le contexte ;
- la décision ;
- les conséquences ;
- le statut ;
- la date ;
- les modules impactés.

## Statuts

- Proposed
- Accepted
- Deprecated
- Superseded

## ADR validées

- ADR-001 — Naming Convention
- ADR-002 — Core Entity: Organisation
- ADR-003 — Ownership Model
- ADR-004 — Workspace Model
- ADR-005 — Product Library
- ADR-006 — Product Strategy
- ADR-024 — Project Driven Architecture
- ADR-025 — Multi-brand Model
